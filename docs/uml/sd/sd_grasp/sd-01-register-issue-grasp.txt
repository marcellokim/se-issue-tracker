SD 01 Register Issue - GRASP 적용 설명

관련 산출물:
- UC1 Register Issue
- SSD 01: registerIssue(projectId, title, description, priority)
- OC-01: Issue 생성, reporter 설정, status=NEW, IssueHistory(CREATED) 기록

적용 GRASP:

1. GRASP Controller
`:IssueTrackingSystem` 객체가 system operation을 받는다. 여기서 IssueTrackingSystem은 MVC 클래스가 아니라 유스케이스 요청을 받는 GRASP Controller 역할의 설계 객체이다.

2. Information Expert
Issue는 status, priority, reporter, reportedDate, history를 가진다. 따라서 기본 priority 적용과 CREATED history 기록은 Issue 책임으로 표현한다.

3. Creator
IssueHistory(CREATED)는 새 Issue에 기록되는 이력이므로 Issue가 생성하고 자신의 history에 추가하는 것이 자연스럽다.

4. Low Coupling / High Cohesion
GRASP Controller가 Issue 필드를 직접 설정하지 않고 `Issue.create(...)`에 의도를 전달한다. 등록 규칙은 Issue에 모이고, 권한/조회/저장 같은 구현 흐름은 SD에서 축약한다.
