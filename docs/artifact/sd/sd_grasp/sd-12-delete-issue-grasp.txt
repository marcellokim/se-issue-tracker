SD 12 Delete Issue - GRASP 적용 설명

관련 산출물:
- UC9 Manage Deleted Issue
- SSD 12: deleteIssue(issueId, comment)
- OC-10: status=DELETED, dependency 제거, STATUS_CHANGED history, 30개 보관 정책

적용 GRASP:

1. GRASP Controller
`:IssueTrackingSystem` 객체가 삭제 관리 system operation을 받는다. 여기서 IssueTrackingSystem은 MVC 클래스가 아니라 삭제 관리 유스케이스를 조정하는 GRASP Controller 역할의 설계 객체이다.

2. Information Expert
Issue는 자신의 status와 관련 dependency를 기준으로 soft delete 사후조건을 실현하는 중심 객체로 표현된다.

3. Creator
DELETED 상태 전이 이력은 Issue의 상태 변경 기록이므로 Issue가 IssueHistory를 생성한다.

4. Protected Variations
deleted issue 저장 구조는 persistence 쪽에서 처리되지만, SD 작성 단계에서는 SQL/transaction보다 soft delete의 개념적 결과를 보여주는 데 집중한다.

5. Low Coupling / High Cohesion
GRASP Controller는 삭제 가능한 상태인지, dependency 제거가 어떻게 저장되는지까지 직접 알지 않는다.
