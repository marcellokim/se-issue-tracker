SD 10 Reopen Issue - GRASP 적용 설명

관련 산출물:
- UC6 Change Issue State
- SSD 10: changeStatus(issueId, targetStatus=REOPENED, comment)
- OC-09: status=REOPENED, assignee/verifier 자동 복원 없음, fixer/resolver 보존

적용 GRASP:

1. GRASP Controller
`:IssueTrackingSystem` 객체가 reopen 요청을 받는다. 여기서 IssueTrackingSystem은 MVC 클래스가 아니라 상태 변경 유스케이스를 조정하는 GRASP Controller 역할의 설계 객체이다.

2. Information Expert
Issue는 reopen 시 보존할 fixer/resolver와 비워둘 assignee/verifier를 알고 있으므로 reopen 책임을 가진다.

3. Creator
reopen comment와 COMMENTED/STATUS_CHANGED history는 대상 Issue의 기록이므로 Issue가 생성한다.

4. Protected Variations
reopen 이후 재작업 담당자는 자동 복원하지 않고 UC5 배정 흐름으로 분리한다.
