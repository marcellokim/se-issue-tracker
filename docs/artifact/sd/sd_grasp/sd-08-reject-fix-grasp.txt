SD 08 Reject Fix - GRASP 적용 설명

관련 산출물:
- UC6 Change Issue State
- SSD 08: changeStatus(issueId, targetStatus=ASSIGNED, comment)
- OC-13: status=ASSIGNED, 기존 assignee/verifier/fixer 유지

적용 GRASP:

1. GRASP Controller
`:IssueTrackingSystem` 객체가 Tester의 reject 요청을 받는다. 여기서 IssueTrackingSystem은 MVC 클래스가 아니라 상태 변경 유스케이스를 조정하는 GRASP Controller 역할의 설계 객체이다.

2. Information Expert
Issue는 현재 status, verifier, assignee, fixer를 알고 있으므로 `rejectFix(...)` 책임을 가진다.

3. Creator
reject comment와 COMMENTED/STATUS_CHANGED history는 해당 Issue의 기록이므로 Issue가 생성한다.

4. Low Coupling / High Cohesion
rejectFix는 상태를 ASSIGNED로 되돌리고 기존 association을 유지하는 데 집중한다. 다른 DEV로 교체하는 일은 UC5로 분리한다.
