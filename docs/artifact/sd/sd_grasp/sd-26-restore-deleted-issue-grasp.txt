SD 26 Restore Deleted Issue - GRASP 적용 설명

관련 산출물:
- UC9 Manage Deleted Issue
- SSD 26: viewDeletedIssues(projectId), restoreIssue(issueId, comment)
- OC-11: DELETED issue를 삭제 직전 NEW/CLOSED 상태로 복구, dependency 자동 복원 없음

적용 GRASP:

1. GRASP Controller
`:IssueTrackingSystem` 객체가 deleted issue 조회와 복구 요청을 받는다. 여기서 IssueTrackingSystem은 MVC 클래스가 아니라 삭제 관리 유스케이스를 조정하는 GRASP Controller 역할의 설계 객체이다.

2. Information Expert
Issue는 자신의 삭제 전이 이력을 통해 복구 대상 상태를 찾는 것으로 표현된다.

3. Creator
restore 상태 전이 이력은 대상 Issue의 history에 기록되므로 Issue가 생성한다.

4. Protected Variations
삭제 시 제거된 dependency를 자동 복원하지 않는 정책을 note로 분리해 복구 흐름과 dependency 재설정 흐름을 구분한다.
