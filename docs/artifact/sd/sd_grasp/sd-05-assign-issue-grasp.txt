SD 05 Assign / Update Issue Assignment - GRASP 적용 설명

관련 산출물:
- UC5 Assign / Update Issue Assignment
- SSD 05: startAssignment, assignIssue, reassignIssue, changeVerifier
- OC-03/04/05/12: 상태별 배정 변경과 이력 기록

적용 GRASP:

1. GRASP Controller
`:IssueTrackingSystem` 객체가 PL의 배정 관련 system operation을 받는다. 여기서 IssueTrackingSystem은 MVC 클래스가 아니라 배정 유스케이스를 조정하는 GRASP Controller 역할의 설계 객체이다.

2. Information Expert
Issue는 현재 status, assignee, verifier, fixer, resolver를 알고 있다. 따라서 상태별 배정/재배정 규칙은 Issue가 수행하는 것으로 표현한다.

3. Creator
ASSIGNMENT_CHANGED와 필요한 STATUS_CHANGED 이력은 Issue의 변화에서 발생하므로 Issue가 생성하고 기록한다.

4. Protected Variations
후보 추천은 별도 UC8로 분리한다. 추천 방식이 바뀌어도 배정 상태 변경 책임은 Issue 쪽에 남는다.

5. Low Coupling / High Cohesion
GRASP Controller는 branch에 맞는 의도 메시지만 보내고, association 변경과 history 기록은 Issue 안에 응집된다.

6. Optional Extension
PL이 배정 사유나 메모를 남기는 댓글은 배정/재배정이 끝난 뒤 선택적으로 수행되는 확장 흐름으로 표현한다.
